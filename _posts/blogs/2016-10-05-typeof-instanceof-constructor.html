These things seemed to be in contradiction. How could n be a type of string, whose constructor is also a string, but is not in fact an instance of a string?

Looking at the MDN docs for instanceof , it also seemed like this shouldn't be true:

object instanceof constructor

With the help of James Porter, I found this helpful StackOverflow post which explains how string literals "are a different kind of object than objects created from within Javascript." So, for example, creating a string literal {% highlight javascript %} var string = 'boop'; typeof string; // 'string' {% endhighlight %} is different from creating a string object using the String function. {% highlight javascript %} var string = new String('boop'); typeof string; // 'object' {% endhighlight %} These two vars, while they both behave as strings as far as we are concerned, are of different instances.

Why is that? JavaScript is a prototype-based language (versus class-based). So, objects aren't instances of a class. Rather, you can only check if an object has another object in its prototype chain somewhere. And since literals are not an object, while a string created by the String constructor is, the string literal is not an instance of String.
